PyMode runs upstream CPython 3.13 compiled to `wasm32-wasi` inside Cloudflare Workers Durable Objects. This works well for most Python code, but the WASM/WASI/DO environment has inherent constraints.

## Platform Constraints

| Constraint | Details |
|------------|---------|
| **No filesystem persistence** | The WASM linear memory is reset per request. Use KV/R2/D1 for storage. |
| **No network sockets** | Standard `socket` module is unavailable. Use `pymode.tcp` and `pymode.http` instead. |
| **No threads** | `threading` is disabled (`--disable-threads`). Use `pymode.parallel` for real parallelism via child DOs. |
| **No subprocess** | `os.system`, `subprocess`, `os.fork` are not available in WASI. |
| **No signals** | Signal handling is emulated (`-lwasi-emulated-signal`), limited functionality. |
| **30s CPU limit** | Per-request CPU time is capped at 30 seconds (CF Workers limit). |
| **128MB memory** | Per-DO memory limit. Large data processing should use `pymode.parallel` to distribute across DOs. |
| **10MB bundle size** | CF Workers' paid-tier limit on compressed deploy. `pymode deploy` keeps the worker bundle at ~6-9.4 MB gz by staging stdlib, variant Python, and user PyPI deps to the Workers Assets binding (separate 25 MiB-per-file budget). Pandas + a typical pure-Python dep set (jinja2, httpx, etc.) lands at 9.4 MB gz. FastAPI + SQLAlchemy + pandas does **not** fit. CF Python Workers (which ships Pyodide as part of workerd, not your bundle) is the working path for those. |
| **Cold-isolate latency** | Variants pay ~25 s on a fresh isolate (~80% of that is `import pandas` itself executing in fresh Python, same cost on native CPython). Warm isolates serve 75-200 ms. PythonDO + 30 s alarm keeps active regions warm, but CF can still spawn new isolates anywhere. If cold start matters for your workload, CF Python Workers' memory snapshots drop the same workload to ~1 s. |

|----------|---------|
| **Encodings** | `encodings`, `encodings.aliases`, `encodings.utf_8`, `encodings.ascii`, `encodings.latin_1` |
| **JSON** | `json` |
| **Regex** | `re` |
| **Collections** | `collections`, `collections.abc`, `functools`, `operator` |
| **Types** | `enum`, `types`, `typing`, `abc` |
| **Core** | `warnings`, `contextlib`, `dataclasses`, `copy`, `copyreg`, `weakref`, `_weakrefset` |
| **Text** | `string`, `textwrap` |
| **Crypto** | `base64`, `hashlib`, `hmac`, `secrets` |

| **Misc** | `keyword`, `reprlib`, `traceback`, `linecache`, `tokenize`, `token`, `csv`, `pprint` |

### Built-in C Modules (compiled into python.wasm)

These are C extension modules linked directly into the WASM binary:

`array`, `_asyncio`, `_bisect`, `_contextvars`, `_csv`, `_heapq`, `_json`, `_pickle`, `_random`, `_struct`, `_zoneinfo`, `math`, `cmath`, `_statistics`, `_datetime`, `_decimal`, `_md5`, `_sha1`, `_sha2`, `_sha3`, `_blake2`, `_codecs`, `_collections`, `errno`, `_io`, `itertools`, `_sre`, `_thread`, `time`, `_typing`, `_weakref`, `_abc`, `_functools`, `_locale`, `_operator`, `_stat`, `_symtable`, `unicodedata`, `pyexpat`, `_elementtree`

### Polyfilled Modules

| Module | Implementation | Notes |
|--------|---------------|-------|
| `binascii` | Pure Python (`lib/polyfills/binascii.py`) | Replaces the C extension. Covers hex, base64, crc32, uu encoding. |
| `socket` | Pure Python (`lib/polyfills/socket.py`) | Routes through `pymode.tcp` host imports. Provides constants, `getaddrinfo`, `create_connection`. |
| `threading` | Pure Python (`lib/polyfills/threading.py`) | No-op locks within a DO. `Thread.start()` spawns child DOs via `pymode.parallel` for real parallelism. |
| `logging` | Pure Python (`lib/polyfills/logging/`) | Threading-free logging that writes to stderr. Full API: `getLogger`, `basicConfig`, handlers. |
| `multiprocessing` | Pure Python (`lib/polyfills/multiprocessing/`) | `cpu_count()` returns 1, exception classes. Use `pymode.parallel` for real parallelism. |
| `ssl` | Pure Python (`lib/polyfills/ssl.py`) | TLS termination handled by CF's edge. Python code doesn't need raw TLS. |
| `zlib` | Pure Python (`lib/polyfills/zlib.py`) | Minimal compression polyfill for packages that import zlib at module level. |

### Not Available

These stdlib modules require OS features that WASI doesn't provide:

| Module | Reason |
|--------|--------|
| `socket` | Polyfilled, routes through `pymode.tcp` host imports. |
| `ssl` | Polyfilled, TLS termination handled by CF's edge. |
| `subprocess` | No process spawning in WASI. |
| `multiprocessing` | Polyfilled, `cpu_count()` returns 1. Use `pymode.parallel` for real parallelism via child DOs. |
| `threading` | Polyfilled, no-op locks within a DO. `Thread.start()` spawns child DOs via `pymode.parallel`. |
| `sqlite3` | C extension not compiled for WASM. Use D1 binding instead. |
| `ctypes` | No dynamic linking for arbitrary shared libraries. |
| `tkinter` | No GUI in Workers. |
| `curses` | No terminal in Workers. |
| `readline` | No interactive terminal. |
| `mmap` | Not available in WASI. |
| `fcntl`, `termios`, `grp`, `pwd` | Unix-specific, not in WASI. |
| `select`, `selectors` | Polyfilled, `selectors` bundled in stdlib. |
| `signal` | Emulated only, limited functionality. |
| `resource` | No resource limits API in WASI. |

## Python Language Support

Full CPython 3.13 language support including:
- All syntax features (match/case, walrus operator, f-strings, etc.)
- Generators, async generators, coroutines
- Decorators, metaclasses, descriptors
- Exception groups, `ExceptionGroup`
- Type hints, `typing` module

**Not supported:**
- `async`/`await` at the handler level (use Asyncify for I/O instead)
- `asyncio` event loop (no I/O multiplexing in WASI)

## Package Support

### C-extension packages: current state

`pymode deploy` auto-detects when your project depends on a C-extension
package and, if a variant wasm exists, uses that variant directly (fast
deploy, no AOT rebuild). The numpy variant is wired and verified
end-to-end as of this writing.

| Package | How it's served | Verified deployable today |
|---|---|---|
| numpy | `python-numpy.wasm` variant | ✅ (linalg stubbed, no LAPACK) |
| Pillow | `python-pillow.wasm` variant | ✅ |
| pydantic_core | `python-pydantic-core.wasm` variant | ✅ (rebuild currently fails on libc `_CLOCK_*_CPUTIME_ID`; pre-built wasm works) |
| pyyaml | pure-Python fallback (uv pip) | ✅ |
| multidict / yarl / frozenlist / propcache | pure-Python fallback (uv pip) | likely (same pattern as pyyaml) |
| msgpack | pure-Python fallback (uv pip) | likely |
| markupsafe | statically linked | ✅ |
| regex / xxhash | statically linked | ✅ |
| ujson | `python-ujson.wasm` variant | ✅ |
| zstandard | `python-zstandard.wasm` variant | ✅ |
| **pandas** | `python-pandas.wasm` variant (numpy + pandas 2.2.3) | ✅ DataFrame/Series ops work; bundle 9.4 MB gz |
| scipy | needs Fortran cross-compile | no |
| scikit-learn | depends on scipy | no |
| lxml | needs libxml2 + libxslt cross-compile | no |
| cryptography | needs OpenSSL cross-compile | no |
| orjson | Rust-only (no pure-Python fallback) | no |
| cffi | runtime C-compile model incompatible with WASM | no |

**numpy caveats**: the variant ships `_multiarray_umath` (array core +
random + FFT) but does not ship `_umath_linalg`. Linear algebra
functions in `numpy.linalg` are stubbed and raise `ImportError` if
called. Everything else (arrays, math, FFT, random) works.

For projects needing scipy / scikit-learn / lxml / cryptography,
**Cloudflare's official Python Workers ships pre-built wheels for those
via Pyodide**, that's the working answer today. Pyodide's runtime is
compiled into workerd itself, so it doesn't eat the user's 10 MiB worker
bundle the way pymode's `python.wasm` does. As of 2026 they also added
memory snapshots that drop pandas-class cold start to ~1s (vs pymode's
~25 s for a cold isolate).

Closing pymode's catalog gap is a multi-month engineering project per
package (scipy needs a Fortran toolchain to wasm; lxml needs libxml2
+ libxslt cross-compile; cryptography needs OpenSSL). Not on the
roadmap. The project is parked.

### Polyfilled C/Rust Extensions

These packages have native extensions replaced with pure-Python polyfills:

| Package | Original | Polyfill |
|---------|----------|----------|
| `uuid_utils` | Rust (`_uuid_utils`) | `stdlib uuid` + `time` + `os.urandom` |
| `xxhash` | C (`_xxhash`) | `hashlib.sha256` as stand-in |
| `ormsgpack` | Rust (`ormsgpack`) | Wraps pure-Python `msgpack` |
| `jiter` | Rust (`jiter.jiter`) | `stdlib json` |
| `markupsafe` | C (`_speedups`) | Pure Python fallback (built-in) |
| `multiprocessing` | C (process model) | Polyfill: `cpu_count()=1`, exception classes |

### Memory budget

A single Durable Object has a 128 MB memory budget. Most pure-Python
packages run comfortably; CPython itself plus a typical handler costs
~20-40 MB depending on imports. Packages that allocate large arrays
(pandas, numpy) reach the limit faster.

The `pymode.parallel` API spawns child DOs for compute parallelism . 
each child has its own memory budget. This pattern can in principle
fan out work that's too large for one DO, but isn't automatic; you
opt in by calling `spawn()` / `gather()` from your handler.

### Won't Work in WASM (any DO)

| Package | Reason |
|---------|--------|
| **tensorflow** | Needs GPU, ~500MB, C++/CUDA runtime |
| **torch** | Needs GPU, ~2GB, C++/CUDA runtime |
| **scipy** | Needs BLAS/LAPACK (Fortran), complex C |

For ML inference, use **Workers AI** (`env.AI.run()`) instead.

## Recommended path for real Python apps on CF

If you got this far reading the constraints. PyMode is honestly **not** the
right tool for most "real Python apps on CF" in 2026. Use:

- **CF Python Workers** (`pywrangler` + `uv`) for the standard web-app shape
  (FastAPI, Pydantic, httpx, pandas-class data work). Their Pyodide runtime
  doesn't count against your bundle, cold start is ~1s with memory snapshots,
  and they ship the full Pyodide package catalog.
- **CF Containers** if you need scipy / sklearn / SQLAlchemy / anything that
  doesn't fit Pyodide. Trade global edge distribution for native Python and
  no size cap. Cold start 2-10 s.
- **pymode** if you specifically want upstream CPython 3.13 (not a Pyodide
  fork), want to self-host the wasm so the runtime isn't locked to CF, or
  want a smaller bundle for narrow API workloads where Pyodide's 25 MB is
  overkill.
