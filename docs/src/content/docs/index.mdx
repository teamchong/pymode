## What this is

PyMode is a small tool that lets you compile a C extension into a wasm Python module. The module runs on Cloudflare Workers via a PyMode runtime, and on anything else that runs wasm32-wasi.

It started as an experiment in running upstream CPython 3.13 directly on CF Workers. As of 2026, CF's official Python Workers handle that case better than PyMode does. Their Pyodide runtime is compiled into workerd so it doesn't eat your 10 MiB worker bundle, they added memory snapshots that drop pandas cold start to about 1 second, and their package catalog covers scipy, sklearn, lxml, and so on.

PyMode is useful where Pyodide isn't, which mostly means custom C extensions. You have a `.c` file that wraps a proprietary algorithm or a niche library. Nobody has upstreamed it to Pyodide. PyMode lets you write a small recipe and produce a wasm module from it.

For Rust extensions PyO3's bridge adds the same per call cost regardless of which runtime hosts it, so PyMode doesn't have an edge there. C extensions are where the value shows up.

## Lessons from the experiment

The runtime works as documented, but the experiment is parked because the shape is wrong for the workload that matters now: AI agents writing Python ad-hoc. Agents want a stateful REPL session — `exec` a code string against live `globals()` — not a `wrangler deploy` round-trip.

The root cause is at the bottom of the stack. PyMode targets `wasm32-wasi`, which has no `dlopen`. Every C extension must be statically linked into `python.wasm` at build time. That forces a variant-profile model (one prebuilt wasm per package combination), which forces a compile step, which forces a deploy round-trip, which kills the REPL shape. Pyodide targets `wasm32-emscripten`, has `dlopen`, lazy-loads `.so` files, and Cloudflare compiled the whole thing into workerd — a moat a third party can't cross from user space.

Don't use PyMode for real work. It's a personal experiment that the post-mortem explains the failure shape of. If you landed here looking for Python on Cloudflare, use CF Python Workers. If you're here because you have one obscure C extension and want to see how the recipe system works, the code still runs.

Full retrospective on GitHub: [`docs/post-mortem.md`](https://github.com/teamchong/pymode/blob/main/docs/post-mortem.md).

## When to use which tool

If your packages are all in Pyodide, use CF Python Workers (`pywrangler` + `uv`). Faster cold start, larger catalog, no need to maintain wasm builds.

If you need scipy, sklearn, SQLAlchemy, or anything beyond Pyodide's catalog, use CF Containers. Trade global edge for native Python.

If you have a C extension that isn't in Pyodide and you don't want to upstream it, PyMode might fit. Same goes for cases where you want the wasm to run outside CF (locally via wasmtime, on Fastly Compute, etc.).

## How it works

```
CF Request
  Worker bundle (in 10 MiB cap):
    python.wasm (CompiledWasm, about 8 MB gz)
    user-files.ts (your .py files inlined)
    thin stubs for stdlib + site-packages

  Workers Assets (25 MiB per file, separate budget):
    stdlib-data.dat.gz
    extension-site-packages.zip.gz
    site-packages.zip.gz

  Per request:
    fetch + gunzip assets, instantiate wasm
    Variant deploys route through PythonDO with alarm based keep alive
    Pure Python deploys run inline in the worker isolate
    on_fetch(request, env) returns a Response
```

Cold isolate latency for variants is about 25 seconds (mostly `import pandas` itself, same cost on native CPython). Warm isolates serve 75 to 200 ms. PythonDO sets a 30 second alarm to keep active regions warm.

## Honest comparison

| | PyMode | CF Python Workers |
|---|---|---|
| Runtime in your bundle | yes, 8 MB gz | no, compiled into workerd |
| Cold start (pandas) | ~25 s | ~1 s with memory snapshots |
| Package catalog | 5 variants plus ~40 pure Python | full Pyodide catalog |
| Custom C extensions | recipe based, you build them | upstream Pyodide PR required |
| Python flavor | upstream CPython 3.13 | Pyodide's CPython fork (Emscripten) |
| Self hostable | yes (any wasm32-wasi runtime) | CF only |
| Active development | no | yes |

If you're picking a tool today and your packages are in Pyodide, use CF Python Workers. PyMode is parked, with the runtime working as documented.
