## About this project

PyMode is a personal experiment in running upstream CPython 3.13 directly on Cloudflare Workers via `zig cc` → `wasm32-wasi` — no Pyodide, no Emscripten, no CPython fork. It went as far as compiling pandas to wasm and deploying it on a Workers Paid plan via the Workers Assets binding.

Where pymode lands in 2026:

- **Cloudflare's Python Workers** ship Pyodide compiled into workerd itself (the runtime doesn't count against your 10 MiB worker bundle). They added memory snapshots that drop pandas-class cold start from ~10 s to ~1 s, and the `pywrangler` + `uv` workflow that handles arbitrary Pyodide-compatible packages. For most "real Python on CF" use cases, that is the working answer.
- **PyMode** stays interesting only where you specifically want upstream CPython, want to self-host the wasm so it isn't locked to CF, or want a smaller bundle for narrow API workloads where Pyodide's 25 MB runtime is overkill.

This site documents how pymode works, what it can run today, and the architectural constraints. It is not actively developed.

## How it works

```
CF Request
  → Worker (in bundle):
      python.wasm (CompiledWasm)
      user-files.ts (your .py files)
      thin stubs for stdlib + site-packages
  → ASSETS binding (off-bundle):
      stdlib-data.dat.gz
      extension-site-packages.zip.gz   ← variant Python layer
      site-packages.zip.gz             ← user PyPI deps
  → Promise.all([fetch+gunzip each, instantiate wasm])
  → Variant deploys: route through PythonDO with alarm-driven keep-alive
  → Pure-Python deploys: run inline in the worker isolate
  → on_fetch(request, env) → JSON-encoded Response
```

Variants stay warm because PythonDO sets a 30 s alarm on every request, well inside CF's ~70 s eviction window. Cold-isolate latency is ~25 s (mostly `import pandas` itself); warm-isolate latency is 75–200 ms.

## Honest comparison

| | pymode | CF Python Workers (Pyodide) |
|---|---|---|
| Runtime in your bundle | yes, 8 MB gz | no — compiled into workerd |
| Pandas-class cold start | ~25 s on a fresh isolate | ~1 s with memory snapshots |
| Package catalog | 5 C-ext variants + ~40 pure-Python | full Pyodide catalog including scipy/sklearn/lxml |
| C extensions | per-recipe; you cross-compile | provided |
| Python flavor | upstream CPython 3.13 | Pyodide's fork (Emscripten) |
| Self-hostable | yes (any wasm32-wasi runtime) | CF-only |
| Maintained | no | yes |

If you'd asked me in 2024, pymode had a clearer story. In 2026 the Pyodide path got significantly better and now wins for typical Python apps on CF. We document this honestly so people don't pick the wrong tool.
